Datenhaltung

  
   
Durch die Verwendung von DS Workbench wird, allein durch den Download der Weltdaten, einiges an Plattenplatz benötigt. Sicher, in Zeiten von Gigabyte und Terrabyte erscheint das verschwindend gering, aber als Nutzer sollte man wohl zumindest einmal gelesen haben, was wann auf die Festplatte geschrieben wird.
Wie bereits erwähnt nehmen anfangs den größten Teil die Weltdaten ein. Dies kann, je nach Spieleranzahl auf dem Server 5 MB ausmachen.

Die Weltdaten werden nicht direkt von den DS Servern geladen, sondern von einem privaten Webspace. So ist es garantiert, dass jeder Nutzer dieselbe Version der Weltdaten hat, was in späteren Versionen von DS Workbench eine wichtige Rolle spielen wird.
Zugleich bedeutet das, dass sich die Möglichkeit eines Weltdatendownload nach der Version richtet, die auf dem Server vorliegt, was den begrenzten Kapazitäten eines Servers sehr zugute kommt.

Alle Daten für einen Server werden im DS Workbench Verzeichnis, in einem Unterverzeichnis server/<SERVER_ID> abgelegt. Dazu zählen auch eurer persönlichen Daten, also erstellte Angriffe oder Markierungen. Wenn ihr mal in ein Serververzeichnis schaut werdet ihr eine Reihe von Dateien mit der Endung XML finden. Einige dieser Dateien enthalten eure erstellten Daten:
 
attacks.xml
 Enthält die erstellten Angriffspläne für den Server
forms.xml
 Enthält die Formen, die mit den Zeichenwerkzeugen erstellt wurden
markers.xml
 Enthält die Farbmarkierungen von Spielern und Stämmen
tags.xml
 Enthält die verteilten Tags (Gruppen) und die zugeordneten Dörfer
troops.xml
 Enthält die importieren Truppen
 
Wie ihr seht können auch diese Daten, abhängig von der Dorfanzahl und der Intensivität der Nutzung von DS Workbench ebenso auf eine gewisse Größe anwachsen. Diese Dateien werden daher nicht direkt bei jeder Veränderung auf die Festplatte geschrieben, sondern erst wenn DS Workbench ordnungsgemäß beendet wird. Sollte DS Workbench über den Taskmanager (Windows) beendet werden oder über den ähnlich drastischen kill Befehl (Linux), so gehen die Daten, die seit dem letzten Start verändert wurden, unwiderruflich verloren.

Alle anderen Dateien die sich im jeweiligen Serververzeichnis befinden gehören zu den Servereinstellungen und den Weltdaten.
 
Sollten aus irgendeinem Grund die auf eurer Platte gespeicherten Weltdaten und/oder Servereinstellungen fehlerhaft und nicht mehr zu lesen sein, beendet DS Workbench und löscht alle Dateien in einem Serververzeichnis, außer natürlich die XML Dateien mit euren persönlichen Daten, die oben aufgelistet sind. Anschließend startet ihr DS Workbench neu und die Serverdaten werden erneut heruntergeladen.
Dieses Vorgehen ist allerdings nur im äußersten Notfall anzuwenden. Wie weiter oben erwähnt, wird es dadurch keine andere/aktuellere Version der Weltdaten geben.
