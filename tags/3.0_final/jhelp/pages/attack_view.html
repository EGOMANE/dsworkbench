Angriffsübersicht
 
 

Die Angriffsübersicht dient der Verwaltung und der Bearbeitung von erstellten Angriffen. Sie ist somit mehr oder weniger der Dreh- und Angelpunkt von DS Workbench. Angriffe sind in DS Workbench in Angriffsplänen organisiert. Beim ersten Start existiert lediglich der Standardangriffsplan mit dem Namen 'default' und ein Angriffsplan 'Manuelle Planung', der mit dem manuellen Angriffsplaner verküpft ist. Das Menü des Angriffsplaners beinhaltet die folgenden Funktionen:

  • Klick-Konto: (siehe unten "Sonderfunktionen")
  • Bearbeiten: (siehe unten "Sonderfunktionen")
  • Bearbeiten
    • Abgelaufene Angriffe löschen: Entfernt alle Angriffe die bereits abgeschickt werden mussten
    • Zeit ändern: Ändert die Zeiteinstellungen für die gewählten Angriffe. Hier besteht die Möglichkeit, Angriffe auf bestimmte Zeitpunkte, um eine bestimmte Dauer oder zufällig zu verschieben.
    • Angriffstyp und/oder Einheit ändern: Erlaubt es nachträglich den Angriffstyp und/oder die Einheit zu verändern. Achtung: Wird die Einheit verändert, so ändert sich auch die Laufzeit.
    • "Übertragen"-Markierung löschen: Entfernt die "In den Browser übertragen" Markierung für die gewählten Angriffe.
    • Angriffe auf der Karte (nicht) einzeichnen: Aktiviert oder deaktiviert das Einzeichnen der gewählten Angriffe auf der DS Workbench Hauptkarte.
  • Übertragen
    • Im Klartext: Überträgt die gewählten Angriffe im Klartextformat in die Zwischenablage. Dieses Format eignet sich dazu, die Angriffe in z.B. Excel einzufügen.
    • Als HTML Datei exportieren: Exportiert alle gewählten Angriffe in eine HTML Datei. Mit dieser Datei ist es dann möglich, einen Überblick über die zu verschickenden Angriffe zu haben, ohne dass DS Workbench geöffnet sein muss. Weiterhin kann aus der HTML Datei heraus direkt der Versammlungsplatz für einen jeden Angriff geöffnet werden.
    • Gewählten Angriff in den ReTimer übertragen: Überträgt den gewählten Angriff in den DS Workbench ReTimer. So können einfach re-time Angriffe aus der Angriffsübersicht heraus geplant werden.
    • Gewählte Angriffe in den Browser übertragen: Überträgt die gewählten Angriffe in den Browser, sofern das Klick-Konto entsprechend viele Klicks aufweist. Ist dies nicht der Fall, werden soviele Angriffe übertragen wie Klicks vorhanden sind. Einzelangriffe können immer übertragen werden.
    • Gewählte Angriffe als Userscript exportieren: (siehe unten "Sonderfunktionen")
  • Sonstiges
    • Standardangriffe einstellen: (siehe unten "Sonderfunktionen")
    • Angriffsbenachrichtigung: Ist diese Funktion aktiviert, wird der Benutzer vor anstehenden Angriffen über ein Popup informiert.
 

Shortcut-Funktionen
 
  • Kopieren per STRG+C: Kopiert alle gewählten Angriffe in einem besonderen Format die Zwischenablage. Von dort aus können die Angriffe in einem anderen Angriffsplan per STRG+V eingefügt werden.
  • Ausschneiden per STRG+X: Kopiert alle gewählten Angriffe in einem besonderen Format die Zwischenablage und löscht sie anschließend aus dem aktuellen Angriffsplan.
  • Einfügen per STRG+V: Fügt Angriffe die vorher per STRG+C oder STRG+X in die Zwischenablage übertragen wurden in den aktuellen Angriffsplan ein. Diese Angriffe können dabei entweder aus der Angriffsübersicht selbst oder aus der Ergebnisübersicht des Angriffsplaners kommen.
  • BB-Code per STRG+B in die Zwischenablage kopieren: Kopiert die gewählten Angriffe als BB-Codes in die Zwischenablage, um sie z.B. im Spiel einzufügen.
  • Löschen per ENTF: Löscht alle gewählten Angriffe.
  • Suchen per STRG+F: Erlaubt eine Suche nach Angriffen. Suchkriterien sind dabei z.B. der Name des Angreifers, Dorf- oder Stammesnamen.
 

Sonderfunktionen

 
  • Klick-Konto: Das Klick-Konto dient dazu, entsprechend der DS-Regeln, für jede Aktion im Spiel einen Klick durchzuführen. Will man mehrere Angriffe auf einmal in einem Tab im Browser öffnen, so muss man zuerst eine entsprechende Anzahl Klicks "sammeln", um anschließend eine entsprechende Anzahl an Angriffen übertragen zu können. Ist das Klick-Konto leer, können nur Einzelangriffe übertragen werden.
  • Profil-Schnellauswahl: Verwendet man DS Workbench sowohl für seinen eigenen, als auch für einen UV-Account, so kann man für beide Accounts ein Profil in DS Workbench erstellen und über die Profil-Schnellauswahl festlegen, über welchen Account die Angriffe verschickt werden sollen, ohne in den Einstellungen das Profil manuell ändern zu müssen. Voraussetzung hier ist, dass der Angriff von einem Dorf ausgeht welches dem Account gehört und dass ein UV-Profil eine korrekte UV-ID besitzt.
  • DS Workbench Angriffs-Info Script:
 
 
DS Workbench bietet die Möglichkeit, Angriffe als Userscript zu exportieren, um sie im Spiel darstellen zu können. Mögliche Orte für die Darstellung sind die Übersichtsseiten (linker Screenshot), wo vor jedem Dorf ein Symbol anzeigt, ob Angriffe abgeschickt werden müssen (grün), alle Angriffe bereits abgeschickt sein müssten (rot) oder ob keine Angriffe aus diesem Dorf geplant sind (braun). An allen anderen Stellen, wie z.B. dem Versammlungsplatz (rechter Screenshot) zeigen Tabellen an, auf welches Ziel Angriffe aus diesem Dorf abgeschickt werden müssen und wann die geschehen muss.
Das Userscript liegt nach der Erstellung im DS Workbench Verzeichnis und trägt den Namen 'zz_attack_info.user.js'. Von dort muss es im Browser installiert werden. Wie die zu geschehen hat ist hier beschrieben.
  • Standardangriffe festlegen:
 
Das Festlegen dient einem weiteren Script-Feature von DS Workbench. Überträgt man Angriffe in den Browser so können, wenn das zweite DS Workbench Userscript im Browser installiert ist und in DS Workbench Truppeninformationen aus dem Spiel importiert wurden, die Einheiten ebenfalls im Spiel eingetragen werden. Hierbei kann man für jeden Angriffstyp separat bestimmen, welche Einheiten geschickt werden sollen. Der obere Screenshot zeigt mehr oder weniger komplexe Beispiele, auf welche Art und Weise man Standardangriffe definieren kann. In den meisten Fällen sollte es genügen, feste Zahlen einzutragen, wie es im Beispiel beim den Angriffstypen 'AG' und 'Fake' der Fall ist.
Die Angriffsübersicht ist eine sehr komplexe Darstellung und kann daher auf einigen Rechnern sehr langsam reagieren. Ist dies bei euch der Fall empfehle ich, die Spalte "Verbleibend", welche den Countdown für jeden Angriff anzeigt, auszublenden. Wie ihr Spalten ausblendet findet ihr hier.
